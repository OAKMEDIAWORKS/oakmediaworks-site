01 — Anchor

Truckee Blue.

A working blue. Lived-in, designer-led, not French-institutional. The single anchor color from which the rest of the system tunes.

Truckee Blue

#4E85DF · --brand-truckee

Sits naturally on warm cream paper; lifts cleanly on coffee dark. The tier is built so the canonical hex appears in both modes — light uses it as --accent, dark preserves it in --accent-deep for fills, halos, and chart bands.

Named for Truckee, California — the Sierra town just over the hill from Lake Tahoe. Switched from Klein Blue (#002FA7) on 2026-05-06: Klein read too clinical against the cream surface. Truckee Blue is warmer, more approachable, still distinctly OAK.

12 — Icons

Lucide. 1.5 stroke.

Standardize on Lucide for every OAK app. Don't mix icon libraries. Stroke 1.5px (override Lucide's default 2px) to match Satoshi's weight. Color inherits from currentColor.

Sizes

14px
inline · dense UI
Tab labels, chip icons, dense table cells
16px
default
Buttons, inline alongside body text
20px
surface UI
Nav rails, card headers, tooltips
24px
prominent
Hero callouts, empty states, large buttons

Use in your app

// React (lucide-react)
import { Search, X, Check } from 'lucide-react';

<Search size={16} strokeWidth={1.5} />
<X size={20} strokeWidth={1.5} />
/* Static HTML — Lucide static SVG, override stroke */
<svg width="20" height="20" stroke-width="1.5"
     stroke="currentColor" fill="none" viewBox="0 0 24 24">
  <path d="..."/>
</svg>

When the system needs an icon Lucide doesn't have, draw it on the same 24×24 grid with 1.5px stroke, round line caps, round line joins. Don't introduce icons from other libraries — they'll fight Lucide's geometry.

13 — How to use

Drop it into any OAK app.

Pull tokens from canon, set the theme attribute, use variables. No hardcoded hex anywhere in your app — that's the whole point.

<!-- In your app's <head> -->
<link rel="stylesheet" href="https://oakmediaworks.com/oak-tokens.css">

<!-- Mode control on the html element -->
<html data-theme="dark">   <!-- explicit dark -->
<html data-theme="light">  <!-- explicit light -->
<html>                     <!-- follow OS preference -->
/* In your component CSS — always reach for tokens, never hex */
.my-card {
  background: var(--bg-surface);
  color: var(--text-1);
  border: 1px solid var(--rule);
  border-radius: var(--radius);
}

.my-link {
  color: var(--accent-deep);   /* AA on cream */
}

.my-button {
  background: var(--accent);
  color: var(--accent-on);
}

If your app needs to vendor the file (offline, build-time bundling), copy oak-tokens.css from the canon repo. When canon updates, re-pull. Don't fork token values into your app — keep this repo as the single source.

14 — Don'ts

Ways to get noticed.

These are the violations that get caught in review. Internalize them.

×
Don't reintroduce Klein Blue. It's retired. The hex is #002FA7 — if you see it in a diff, replace it.
×
Don't reintroduce green to brand surfaces. Greens were systematically removed from landing/pricing in favor of the brand blue. Semantic green stays for success states only.
×
Don't reach for pure grays. Warm coffee tones only. Cool grays on warm surfaces look broken.
×
Don't put body links on --accent. Use --accent-deep on cream — AA-compliant for normal text. Reserve --accent for fills and ≥14px semibold.
×
Don't use #000 or #FFF on any surface. The text and surface tiers handle both modes correctly — pure black and pure white break the temperature.
×
Don't hardcode hex values in app CSS. Always reach for the variable. If the token doesn't exist, the token system needs to grow — file an issue against this repo.
